# OptimusKG: Unifying biomedical knowledge in a modern multimodal graph

## TL;DR
Presents OptimusKG, a comprehensive biomedical knowledge graph integrating 190K+ nodes across molecular, anatomical, clinical, and environmental domains from 18 ontologies. Built with ontology-harmonized schema, property-rich metadata, FAIR principles, and evidence provenance to enable graph-based machine learning, LLM grounding, and biomedical discovery (e.g., hypothesis generation).

## The problem (in plain language)
Biomedical knowledge graphs (KGs) exist widely but suffer from fragmentation: many lack standardized schema, preserve unstructured data only, lack provenance tracking, have outdated snapshots, and don't provide structured metadata. This limits interoperability and makes them unreliable for LLM grounding, reproducible analysis, or quality assessment.

## What they propose
**OptimusKG**: A multimodal labeled property graph (LPG) built from structured + semi-structured biomedical resources via a modular data pipeline (Landing → Bronze → Silver → Gold layers following medallion architecture):

- **Unified schema**: Biolink Model (v3.2.1) as base ontology, extended with Gene Ontology, Uber Anatomy Ontology for molecular/anatomical/clinical coverage
- **Granular metadata**: Type-specific properties, cross-references, provenance across 150 property keys
- **Quality validation**: Schema constraints, entity harmonization, literature-based evidence verification using AI agents
- **FAIR principles**: Standardized identifiers, version control, full reproducibility

Contains 190,531 nodes (10 types), 21,813,816 edges (26 relations), 67M+ property instances encoding 110M+ values. Distributed as Apache Parquet for ML/vector retrieval.

## Key results
- **Scale**: 190K+ nodes, 26 relation types, 150 property keys across molecular/anatomical/clinical/environmental domains
- **Evidence validation**: 70% of edges have literature support; 83.4% of sampled false edges lack support (quality check)
- **Comprehensive coverage**: Integrates PrimeKG, BioKG, CROssBARanatomical, clinical, drug, disease-specific ontologies
- **Reproducibility**: Version-controlled pipeline enables edge-case debugging and systematic update cycles

## What's reusable in Linus
**Phase 2-3 relevance** (KB integration): OptimusKG's medallion architecture (Landing → Bronze → Silver → Gold) and schema harmonization workflow provide a direct template for Dan's KnowledgeBase v1 pipeline. The FAIR principles and provenance tracking are directly adoptable for Linus's knowledge integration layer. The evidence-grounding pattern (scientific literature verification) aligns with Phase 3 agent-based fact-checking workflows.

**Phase 3+ (Knowledge & Parallel Agents)**: OptimusKG's type-specific property design and cross-domain ontology harmonization are reusable patterns for Linus's domain-specific tools (genomics, biochemistry). The graph-based retrieval for LLM grounding is applicable to Linus's future knowledge-grounded inference.

## What's NOT applicable / hype filter
- Biomedical-specific domain. Linus must be domain-agnostic initially, though Dan's scientific expertise means biomedical grounding will eventually be a strength
- Requires large-scale integration of 18+ external ontologies; Linus will start with Dan's paper corpus + notes
- Apache Parquet serialization; Linus may use SQLite or alternative graph formats given Apple Silicon constraints
