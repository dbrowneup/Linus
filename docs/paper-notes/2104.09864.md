# RoFormer: Enhanced Transformer with Rotary Position Embedding

## TL;DR

Su et al. replace Vaswani's sinusoidal positional encodings with Rotary Position Embedding (RoPE): a multiplicative
rotation applied directly to query and key vectors via a rotation matrix. RoPE encodes absolute position information
while incorporating explicit relative position dependency into the self-attention formulation. The key insight is
geometric: position is encoded as a rotation angle in 2D planes (one plane per pair of embedding dimensions), making
relative position decay naturally with distance. This substitution is foundational to every modern LLM Worker model
(Qwen, Mistral, Llama, etc.) and directly improves long-context modeling and extrapolation.

## The problem (in plain language)

Vaswani's sinusoidal positional encodings work, but they have a limitation: learned models struggle to extrapolate to
sequence lengths longer than those seen during training. The sinusoidal pattern doesn't enforce the relative position
structure the authors intended. Additionally, relative position encoding approaches from Shaw et al. and followers add
computation and parameters without clear clarity on how to integrate them cleanly with self-attention. RoPE asks: can
we encode position multiplicatively (via rotation) such that relative distance naturally decays without extra terms?

## What they propose

Position information is encoded by rotating the query and key embeddings by a rotation matrix that depends on absolute
position m and n respectively:

$$f_q(x_m, m) = (W_q x_m)e^{im\theta}, \quad f_k(x_n, n) = (W_k x_n)e^{in\theta}$$

where $\theta = \theta_0 = 10000^{-2(i-1)/d}$ (inherited from sinusoids). The inner product of rotated query and key
then naturally encodes relative position as a rotation angle:

$$q_m^T k_n = (R^d_{\Theta, m} W_q x_m)^T (R^d_{\Theta, n} W_k x_n) = x_m^T W_q^T R^d_{\Theta, m-n} W_k x_n$$

where $R^d_{\Theta, m}$ is a block-diagonal rotation matrix. The core property: relative position (m - n) appears in
the exponent. In 2D, this is a simple rotation by angle. In higher dimensions, the space is divided into d/2 2D planes,
each rotated independently. Long-term decay is built in: $\theta_i = 10000^{-2(i-1)/d}$ ensures that for large relative
distances, the inner product decays smoothly.

## Key results

On long-context benchmarks (Chinese legal documents, open-domain QA, text classification):

- RoFormer consistently outperforms baseline Transformers with sinusoidal or learned positional encodings.
- Extrapolation to unseen lengths is stronger: models trained on length L generalize better to 2L or 4L.
- Relative position encoding is modeled cleanly without adding separate trainable embeddings or decomposition terms.
- RoPE is already integrated into Huggingface Transformers (standard in modern LLMs).

## What's reusable in Linus

RoPE is non-negotiable infrastructure for every Worker model Linus will run:

- **Qwen2.5-Coder, Mistral-7B, Llama 3.x** all use RoPE, not sinusoids. Any Linus fine-tuning or LoRA work assumes
  RoPE is the baseline.
- **Long-context inference on M1 Max**: RoPE's relative distance decay is critical for KV cache efficiency. The scaling
  base $\theta_0 = 10000$ can be adjusted for longer extrapolation (e.g., $\theta_0 = 100000$ in Llama 3.1 for 128K
  context). Understanding RoPE is a prerequisite for tuning context budgets on the 32 GB M1 Max.
- **BitNet quantization**: The rotary matrix multiplication is element-wise, making RoPE quantization-friendly compared
  to additive sinusoids or learned embeddings. BitNet 1.58-bit models (in the Linus BitNet thread) rely on RoPE's
  multiplicative structure.
- **Streaming inference (mlx-flash)**: RoPE is applied per-token, not per-layer, so streaming weight loads interact
  cleanly with position encoding. No need to recompute RoPE for past tokens; only the current token gets rotated.

## What's NOT applicable / hype filter

The specific 2D case (Section 3.2.1) is pedagogical; modern implementations use vectorized block-diagonal multiplication
(Eq. 14–15) and are hardware-optimized in Metal. The paper's experiments are on Chinese NLU tasks; Linus tasks are
scientific/code-oriented, but the architectural benefit is universal.

The long-term decay constant $\theta_0 = 10000$ is a choice point: modern models scale it up for longer context windows.
This is a knob to tune per-Worker, not a fixed parameter.

RoPE is not applied to the Value vectors in standard implementations, though the paper briefly mentions variants. Modern
implementations (Qwen, Llama, Mistral) apply RoPE to Q and K only.
