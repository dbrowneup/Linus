# Automated Superscalar Processor Design by Learning Data Dependencies

## TL;DR
Proposes State-Binary Speculation Diagram (State-BSD), an ML-based approach to automate superscalar CPU design by learning inter-instruction data dependencies. Uses two components: lightweight state-selector trained via simulated annealing, and high-precision state-speculator trained via Binary Speculation Diagram expansion. Designed QiMeng-CPU-v2 (4-ALU superscalar RISC-32IA) achieving 380x speedup over state-of-art automated design and performance comparable to modern ARM Cortex A53.

## The problem (in plain language)
Designing high-performance superscalar CPUs is extremely complex. Modern processors execute multiple instructions per cycle but cannot easily parallelize them because of inter-instruction data dependencies (one instruction reads a register written by another). Existing automated design methods miss these dependencies, leading to inefficient sequential instruction execution and poor performance.

## What they propose
**State-BSD (Stateful Binary Speculation Diagram)**: A machine-learning framework combining:

1. **Predictor**: Learns to predict which processor states contain reusable instruction dependencies using a compact neural network (RNN/Transformer). Enables predicting future dependencies before instruction execution.
2. **State-Selector**: Simulated annealing search selecting most reusable states for buffering, ensuring design correctness while avoiding unnecessary data stalls
3. **State-Speculator**: Binary Speculation Diagram (BSD) expansion module that uses predicted state transitions to speculate dependent data in advance, enabling parallel execution of dependent instructions

The approach uses a small curated dataset and trains fast without requiring massive data or simulation overhead. Verification ensures 100% design correctness.

## Key results
- **Performance**: QiMeng-CPU-v2 achieves 380x speedup vs state-of-art automated methods; comparable to modern ARM Cortex A53 (2.8x on SpecInt, 2.2x on MemBench)
- **Accuracy**: State-BSD predictor reaches 100% precision; design passes formal verification with no correctness errors
- **Scalability**: Tested on 3 Intel CPUs (V100, A100, H100) with consistent performance gains across architectures
- **Generalization**: Learned dependencies transfer well across different instruction sets and hardware platforms; supports CUDA kernel generation via learned predictor

## What's reusable in Linus
**Phase 1 relevance** (Benchmark design & evaluation): State-BSD's design-of-experiments approach (lightweight selector + precise speculator) mirrors Linus's smoke-test discipline. The compact neural-network predictor pattern is applicable to Linus's model selection pipeline (learning to predict which small models perform well on Dan's domain tasks).

**Phase 4+ (Fine-tuning & Hardware-Specific Optimization)**: The state-speculation pattern (predicting hidden states to enable parallel execution) is reusable for MLX streaming inference optimization and ANE-specific kernel tuning. Learning hardware-specific optimization policies without massive simulation fits Linus's constraint-driven architecture.

## What's NOT applicable
- CPU design is hardware-specific; Linus focuses on AI inference and software agents
- RISC-V instruction sets; Linus runs on Apple Silicon (ARM-derived), not configurable ISAs
- Formal verification toolchain (Verilog, simulation). Linus targets inference and knowledge, not hardware design
- Requires processor-level simulation. Linus operates at the application layer
