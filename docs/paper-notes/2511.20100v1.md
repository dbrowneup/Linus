# QiMeng-Kernel: Macro-Thinking Micro-Coding Paradigm for LLM-Based High-Performance GPU Kernel Generation

## TL;DR
Proposes MTMC (Macro Thinking Micro Coding), a hierarchical LLM-based framework that decouples GPU kernel generation into semantic-level optimization strategies (Macro Thinking) and stepwise low-level code implementation (Micro Coding). Achieves 100% accuracy and 70% speedup over general-purpose LLMs on KernelBench, with 7.3x speedup over hand-optimized PyTorch Eager kernels.

## The problem (in plain language)
Developing high-performance GPU kernels is critical for AI but extremely challenging. LLMs struggle because they need to understand both high-level hardware characteristics and low-level implementation details simultaneously, making it hard to generate correct and efficient kernels. Existing approaches fail on CUDA/Triton implementations and lack comprehensive hardware-specific optimization knowledge.

## What they propose
**Macro Thinking Micro Coding (MTMC)** framework with two key components:

1. **Macro Thinking**: Lightweight LLMs generate semantic optimization actions (tiling, fusion, pipelining, reordering) using domain-specific knowledge and hardware characteristics, decomposing complex optimization into interpretable steps
2. **Micro Coding**: General-purpose LLMs iteratively implement these steps as GPU code, with RL-trained policy to guide optimization and rule-based reward shaping to maximize correctness

The pipeline decouples optimization strategy from implementation, enabling efficient policy learning on a compact human-curated dataset without prohibitive data requirements.

## Key results
- **KernelBench**: 100% accuracy, 70% speedup vs general-purpose LLMs; 1.2x over 50% speedup on SOTA; 1.3x over domain-specialized LLMs
- **TritonBench**: 96% accuracy, 34x speedup over baselines
- **Hardware platforms**: Evaluated on Intel CPUs (V100, A100, H100) with consistent generalization
- **Comparison**: MTMC is only approach approximating expert-optimized PyTorch Eager kernels while maintaining correctness

## What's reusable in Linus
**Phase 2-3 relevance**: MTMC's hierarchical decomposition (semantic → implementation) is directly applicable to Linus's worker orchestration. The macro-level action space and micro-level code generation pattern maps onto Linus's tool registry and agent spawner. The semantic optimization action space design (tiling, fusion, pipelining) offers a template for specifying domain-specific agent tasks without full implementation details.

## What's NOT applicable
- GPU-specific (CUDA/Triton). Linus runs on Apple Silicon with Metal/MPS and ANE, not NVIDIA GPUs
- Requires large-scale offline dataset and RL training infrastructure; Linus targets lightweight single-worker setups
- Hardware profiling assumes NVIDIA ecosystems; MLX ecosystem profiling would need porting
