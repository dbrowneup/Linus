## Lessons learned (write-back candidates)

### CLAUDE.md / memory (mostly done)

- **Inspect conda envs with `conda list -n <env>`, never `conda run -n <env> python/pip`.** `conda
  run` silently executed the **linus** python (PATH leak from how this shell was initialized) and
  falsely reported the `papers` env was missing umap/bertopic/hdbscan — it has them. (New feedback
  memory candidate; cost two wrong diagnoses.)
- **Submodule-cd trap** (already in `feedback_submodule_cd_trap` memory): bare `git`/`pytest` after
  `cd` into `modules/` (a submodule) operates on the wrong repo — a commit landed on `main` via the
  wrong path and pytest collected 0. Use `git -C <root>` + absolute paths. Worth promoting to
  CLAUDE.md.
- **Subagent Bash is blocked in this env:** the C2 worker did edits in its worktree but couldn't
  git; Maestro lands worktree edits via the recovery pattern (branch in worktree → prettier →
  commit → push → PR → dismantle).
